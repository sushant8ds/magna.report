\chapter{SYSTEM DESIGN}

\justify

This chapter describes the architecture and design of the HPDC Defect Detection System, covering the RF-DETR model architecture, class structure, data flow, and dataset schema.

\section {Architecture of the system (explanation)}
\justify

The system architecture consists of three major components: the Data Engineering Pipeline, the RF-DETR Detection Model, and the Evaluation and Inference Engine. RF-DETR Medium uses a DINOv2 ViT-B backbone, a multi-scale projector neck, and a multi-scale deformable DETR decoder producing end-to-end detections without NMS.

\section {Class Diagram (with brief explanation)}
\justify

The system is organised into four primary Python classes. \texttt{PatchGenerator} handles sliding-window extraction and Shapely polygon clipping. \texttt{DatasetBuilder} applies balancing and augmentation. \texttt{RFDETRTrainer} wraps the training loop with EMA tracking. \texttt{DefectDetector} handles inference and coordinate mapping.

\section {ER diagram  and schema (if applicable) (if applicable with brief explanation)}
\justify

The system does not use a relational database. Data is stored as COCO-format JSON files and PNG patches. The logical schema is:
