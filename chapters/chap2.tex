\chapter{SOFTWARE REQUIREMENT SPECIFICATION}

\justify

This chapter formally specifies the functional and non-functional requirements of the HPDC Casting Surface Defect Detection System. The system ingests high-resolution HPDC casting images, applies a patch-based deep learning pipeline, and outputs bounding box detections annotated with class labels and confidence scores for Crack and Cold Shut defects.

\section{Overview of SRS}
\justify

The Software Requirements Specification (SRS) defines what the system must do and the constraints under which it must operate. The system has two primary stakeholders: Magna International (end user requiring production-scale defect detection) and the university research team (developers responsible for model training, evaluation, and delivery). The SRS covers the Phase 1 scope: detection and localisation of surface defects using bounding boxes.

\section{Requirement Specifications}
\justify

The system is decomposed into a data engineering pipeline, a model training framework, and an inference engine. The requirements below govern all three components.

\subsection{Functional Requirements}

\begin{itemize}[leftmargin=2em]
    \item \textbf{FR1 --- Image Ingestion:} The system shall accept high-resolution HPDC casting images (up to 4096$\times$3000 pixels) in PNG or JPEG format as input.
    \item \textbf{FR2 --- Patch Generation:} The system shall divide input images into overlapping patches using a configurable sliding-window strategy (patch size: 768 or 1024 px; overlap: 20\%--50\%).
    \item \textbf{FR3 --- Annotation Clipping:} The system shall accurately clip COCO polygon annotations to patch boundaries using polygon-aware geometry (Shapely library), recomputing bounding boxes from clipped polygons.
    \item \textbf{FR4 --- Dataset Balancing:} The system shall support configurable downsampling of background (negative) patches to achieve a target positive:negative ratio.
    \item \textbf{FR5 --- Augmentation:} The system shall support selective data augmentation applied only to defect-containing patches (horizontal flip), with automatic bounding box coordinate recomputation.
    \item \textbf{FR6 --- Model Training:} The system shall fine-tune an RF-DETR Medium model on the generated patch dataset, supporting configurable batch size, epochs, input resolution, and worker count.
    \item \textbf{FR7 --- Checkpoint Management:} The system shall maintain both a regular checkpoint and an EMA (Exponential Moving Average) checkpoint during training, saving the best-performing checkpoint on the validation set.
    \item \textbf{FR8 --- Evaluation:} The system shall compute and report COCO-standard metrics: mAP50-95, mAP50, and per-class AP (Crack, Cold Shut) on the validation and test splits.
    \item \textbf{FR9 --- Inference:} The system shall perform patch-based inference on full HPDC images, producing bounding box detections with class labels and confidence scores.
    \item \textbf{FR10 --- Licence Compliance:} The system shall use only model architectures and libraries under licences compatible with commercial deployment (Apache 2.0 preferred).
\end{itemize}

\subsection{Non-Functional Requirements}

\begin{itemize}[leftmargin=2em]
    \item \textbf{NFR1 --- Performance:} Inference on a single 1024$\times$1024 patch shall complete in under 20~ms on a GPU (T4-class or better), supporting real-time production throughput.
    \item \textbf{NFR2 --- Accuracy:} The system shall achieve a minimum EMA mAP50-95 of 0.20 on the test set to be considered production-viable for Phase 1.
    \item \textbf{NFR3 --- Licence Compliance:} All model weights, training frameworks, and key libraries shall use licences compatible with commercial deployment (Apache 2.0, MIT, BSD).
    \item \textbf{NFR4 --- Reproducibility:} All experiments shall be reproducible from documented configurations (patch size, overlap, balancing ratio, augmentation, epochs, batch size).
    \item \textbf{NFR5 --- Scalability:} The patch generator shall handle any input image resolution above 1024$\times$1024 without code modification.
    \item \textbf{NFR6 --- Maintainability:} The pipeline shall be modular (separate patch generation, training, and inference scripts) to allow independent updates.
    \item \textbf{NFR7 --- Security:} Model checkpoints and proprietary HPDC images shall not be distributed publicly and shall remain under Magna International's data governance policy.
    \item \textbf{NFR8 --- Usability:} The inference module shall produce annotated output images with bounding boxes overlaid, requiring no additional post-processing by the quality engineer.
\end{itemize}
